06 Usage Rules

Do's

✅ Always use --bg-primary (#111827) or darker for backgrounds — it's navy, never #000000
✅ Use gold (#F4BB30) sparingly — it's the accent, not a background color
✅ Maintain text hierarchy: primary → secondary → muted → subtle
✅ Use IBM Plex Mono for all data values, metrics, code, and token references
✅ Use teal (#145c80) for analytical/strategic UI elements distinct from brand gold
✅ Prefer subtle borders (--border-default at 6% opacity) over hard lines
✅ Use IBM Plex Sans weight 500-700 for headings, Inter weight 400-500 for body
✅ Animate transitions with --ease-default — never linear or jarring
✅ Semantic colors relate to the palette: success=teal-green, warning=gold, error=coral, info=steel-blue

Don'ts

❌ Never use pure black (#000000) as background — always dark navy
❌ Never use serif fonts — the brand is clean and geometric
❌ Don't mix gold and teal in the same component — pick one accent per element
❌ Don't use Inter, Roboto, or Arial — they're generic and off-brand
❌ Don't exceed 4 text hierarchy levels in one card
❌ Don't use colored backgrounds for cards — stick to bg-secondary/elevated
❌ Don't use hard box-shadows — prefer subtle, dark, blurred shadows
❌ Don't use bright white (#ffffff) for text — use #f8f8f8 (slightly warm)
❌ Don't use generic semantic colors (green/red/blue) — use the theme-aligned versions
